# AI费曼教室

把知识讲给 AI 学生，发现自己哪里没讲明白。

AI费曼教室是一个面向理工科学生的学习工作台。用户先输入学科，并在“知识点讲解 / 题目讲解”两种模式中选择一种；进入讲解工作区后，反复向 AI 学生讲解。系统不打分，只指出讲解中的概念漏洞、逻辑跳跃、公式条件缺失、直观含义不清和表达含混处，讲解结束后生成诊断报告，并可存入左侧对话记录。

## 运行方式

当前版本已经接入智谱大模型，需要通过本地服务运行：

```powershell
npm run dev
```

然后访问 `http://localhost:5173`。

如果只是打开 `index.html`，页面仍能加载，但大模型请求需要本地服务在 `5173` 端口运行。

## 环境变量

项目会从 `.env` 读取智谱配置：

```powershell
ZHIPU_API_KEY=your-zhipu-api-key
ZHIPU_MODEL=glm-5.1
ZHIPU_VISION_MODEL=glm-4.5v
MAX_PDF_BYTES=15728640
PORT=5173
```

`.env` 已被 `.gitignore` 忽略，不要提交真实 API key。

## 当前实现

- 左侧对话栏：新对话、对话记录和反馈入口；调研后台不在用户端展示，可直接访问 `/admin.html`。
- 三阶段学习闭环：创建讲解、讲给 AI 学生、诊断报告。
- 支持“知识点讲解”和“题目讲解”。
- 讲解工作区顶部会展示用户输入的当前知识点或题目，方便讲解时随时核对原题。
- 讲解输入区提供公式输入按钮，可用科学键盘生成 LaTeX 并插入讲解。
- 支持直接上传或粘贴图片/PDF 学习材料。可复制文字 PDF 会走文本理解；扫描版/图片型 PDF 会自动渲染页面并用视觉模型理解，不要求用户手动转成文字版。
- 通过本地后端代理调用智谱大模型，前端不直接暴露 API key。
- 支持多轮补充讲解，并记录问题是否被后续解释修正。
- 最终报告包含讲解主题、理解漏洞、逻辑跳跃、已讲清楚部分、仍需复习问题和下一轮重讲任务。
- 会把当前会话自动保存到左侧对话记录，未生成报告也可继续进入；右键左侧记录可删除。
- 反馈支持上传或粘贴多张截图，并会同步提交时的完整对话记录，便于在后台定位问题。

## 大模型接入位置

当前 `server.js` 提供四个核心 AI 接口：

- `POST /api/chat`：根据用户最新讲解生成 AI 学生追问和漏洞观察。
- `POST /api/report`：根据完整对话生成最终诊断报告。
- `POST /api/prepare-material`：统一理解图片或 PDF 学习材料，生成 AI 学生会话所需的材料上下文。
- `POST /api/recognize-image`、`POST /api/recognize-pdf`：旧版识别接口，保留用于兼容。

输出仍应避免评分，保持“认真、有基础、但不会装懂的 AI 学生”口吻。
